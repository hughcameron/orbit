## Edge-case probing

1. **Forked reviewer silently writes to the default path instead of the brief path.** The spec anticipates this: both review-spec/SKILL.md and review-pr/SKILL.md now state in the "Output path" section that writing to the default path when a brief path was supplied "will cause drive to report the review as missing and trigger a retry." Drive's §5.4.1 retry path kicks in on the missing file, and the second fork receives the same brief; if it also silently writes to the default path, drive escalates with "review could not be completed after 2 forked attempts at <stage>" and stops. Honest failure, no silent success. Verified: the brief templates in §5.3/§7.3 end with "this path takes precedence over the default path in the skill" explicitly to counter this drift.

2. **`review_cycles.review_spec == 3` on drive entry at resumption.** Covered by drive/SKILL.md §5a "Resumption case" and §11 step 4. On resumption with counter at 3 and no synthetic-BLOCK history entry, drive synthesises the BLOCK inline and routes into §8 NO-GO handling *without launching a fork*. The condition explicitly distinguishes "budget-exhausted synthetic BLOCK was not yet written to history" from the already-recorded case. Sound — the only loophole would be a resumption where the synthetic BLOCK *was* recorded but drive.yaml status was not advanced; §8 step 1 writes history before incrementing iteration, so a session death between those writes leaves drive in the correct state to recover via the existing NO-GO handling.

3. **Counter-reset on iteration.** §8 step 3 writes `review_cycles: {review_spec: 0, review_pr: 0}` and nulls `review_cycle_dates` when incrementing iteration. ac-09's "reset to 0 when drive transitions to a new iteration" is satisfied.

4. **session-context.sh with malformed new-shape drive.yaml.** Not tested exhaustively, but the hook uses tolerant `grep || true` parsing and does not read `review_cycles`, so field-level corruption in the new block cannot break the hook. Confirmed by simulation.
