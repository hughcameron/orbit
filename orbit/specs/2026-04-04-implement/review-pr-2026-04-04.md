## Findings

### [MEDIUM] Trailing double quote not stripped from quoted YAML constraints
**Category:** bug
**Description:** When a constraint is written as `- "First constraint in double quotes"`, the awk parser strips the leading `"` via `sub(/^"/, "", line)` but never strips the trailing `"` on the same line. The `sub(/"$/, "", cont)` only operates on continuation lines, not the initial line.
**Evidence:**
```
$ echo '  - "First constraint in double quotes"' | awk '...'
First constraint in double quotes"
```
Test output: `- First constraint in double quotes"` (trailing quote present)
**Recommendation:** Add `sub(/"$/, "", line)` after the leading-quote strip in the `^  - ` block. Specifically, after line 60 (`sub(/^"/, "", line)`), add: `sub(/"$/, "", line)`.

### [LOW] Single-quoted YAML strings not handled
**Category:** edge-case
**Description:** YAML single-quoted strings (`- 'constraint text'`) pass through with quotes intact. The awk parser only handles double-quote stripping.
**Evidence:**
```
Input:  - 'Single quoted: constraint'
Output: - 'Single quoted: constraint'
```
**Recommendation:** Add single-quote stripping alongside double-quote stripping. However, the real spec files in this repo use bare (unquoted) YAML strings, so this is low severity. A defensive fix would add `sub(/^'/, "", line); sub(/'$/, "", line)` alongside the double-quote handling.

### [INFO] "Latest spec" selection is alphabetical, not chronological
**Category:** edge-case
**Description:** When multiple specs share the same date prefix (as in this PR: `2026-04-04-distill`, `2026-04-04-implement`, `2026-04-04-memos`), `sort -r | head -1` picks alphabetically last (`memos`), which may not be the most relevant in-flight spec.
**Evidence:** `find specs -maxdepth 1 -type d -name '20*' | sort -r | head -1` returns `orbit/specs/2026-04-04-memos`
**Recommendation:** This is pre-existing behavior, not introduced by this PR. Noting for awareness only.

## Honest Assessment

This PR delivers what the spec asks for: a SKILL.md that instructs implementing agents to read the spec first, a progress tracking template, and a SessionStart hook that surfaces hard constraints from in-flight specs. The core functionality works -- session-context.sh correctly extracts and displays constraints from real spec files, handles multi-line YAML, empty blocks, missing directories, and EOF gracefully. The one real bug is the trailing double-quote on quoted YAML constraints, which should be fixed before merge since specs authored by different people may use YAML quoting. The single-quote issue is lower priority but trivial to fix at the same time. I recommend REQUEST_CHANGES for the quoting bug fix, after which this is ready to merge.
