**Description:** `.orbit/cards/0008-consolidated-orbit-artefact-folder.yaml` contains 9 lines mentioning bare `cards/`, `specs/`, `decisions/`, `discovery/` paths in its scenario `then:` clauses, `goal:`, `references:`, and `source_lines:` fields. However, ac-13's allow-list literally enumerates: (a) this spec's review artefacts, (b) CHANGELOG.md, (c) the prose-enforcement memo, (d) setup SKILL.md brownfield detector. ### [LOW] F2 — session-context.sh aborts mid-execution if `orbit/` exists but `.orbit/specs/` is absent
**Category:** edge-case
**Description:** When `orbit/` is present without an `.orbit/specs/` subdirectory, the script exits with code 1 (failure) part-way through, before reaching the constraints/state-summary section. Root cause: under `set -euo pipefail`, the pipeline `find .orbit/specs -maxdepth 2 -name 'drive.yaml' 2>/dev/null | head -1` returns non-zero when `.orbit/specs` doesn't exist (find errors on missing dir, head exits 0 with empty input — but pipefail propagates find's failure).
**Evidence:**
```
$ mkdir -p /tmp/sc-test-orbit/orbit
$ cd /tmp/sc-test-orbit && bash plugins/orb/scripts/session-context.sh; echo "exit=$?"
exit=1

$ (set -euo pipefail; find .orbit/specs -maxdepth 2 -name 'drive.yaml' 2>/dev/null | head -1); echo "rc=$?"
rc=1
```
The same pattern applies later at L173 (`find .orbit/specs -maxdepth 1 -type d -name '20*' ...`) and L177 (`find .orbit/cards -maxdepth 1 -name '*.yaml' ...`). All would abort if their target dir is absent.
**Recommendation:** Add a guard `if [[ -d .orbit/specs ]]; then ... fi` around the find pipelines, or change the find invocations to `find .orbit/specs/ -maxdepth 2 ... 2>/dev/null || true | head -1` so the pipeline cannot fail. Greenfield-post-setup is unaffected (setup creates .orbit/specs), and the actual repo's hook works fine — but a manually-created `orbit/` (without subdirs) silently breaks the SessionStart hook. Real-world impact is small but the failure mode is invisible to the user. Minor reliability nit.
