## Findings

### [MEDIUM] ac-04 skill prose overstates the hook's stage-reading behaviour
**Category:** doc-accuracy
**Description:** ac-04's description and `plugins/orb/skills/drive/SKILL.md` §"Halt-temptation guard" both assert that *"the hook reads the current pipeline stage from `drive.yaml` and treats stage-cross widening as a violation of question 3"*. The shipped hook (`plugins/orb/hooks/three-question-test.sh`) does not parse `drive.yaml`'s `stage` field; it only checks for the *presence* of any `.orbit/specs/*/drive.yaml`. The question-3 stderr text mentions *"for the current stage"* but no mechanical check is performed. Doc-type AC close-evidence is satisfied (the prose exists), but the prose itself misrepresents what the hook does — exactly the substrate-engagement failure mode this card was built to prevent.
**Evidence:**
- `plugins/orb/hooks/three-question-test.sh:43-46` — the only `drive.yaml` operation is `drive_yamls=(.orbit/specs/*/drive.yaml)` then `[ "${#drive_yamls[@]}" -gt 0 ]`. No `yq`/`grep` for the `stage:` key, no per-spec drive.yaml selection.
- `plugins/orb/skills/drive/SKILL.md` §"Halt-temptation guard" claim: *"The hook reads the current pipeline stage from `drive.yaml` and treats stage-cross widening as a violation of question 3"*.
- `.orbit/specs/2026-05-19-act-when-authorised/spec.yaml` ac-04 final clause: *"The D1 hook reads the current pipeline stage from `drive.yaml` and treats stage-cross widening as a violation of the authorisation question."*
- Interview D4 framing was internally consistent with this claim: *"reads current pipeline stage from `drive.yaml` and matches against the halt-condition's stage."*
**Recommendation:** Pick one (operator's call, not blocking):
1. Soften the prose + AC text so the hook's role is "asks the question, names the substrate sources; the agent does the stage-match" (matches what shipped). This is the cheapest fix and is honest about what the hook does — `halt-conditions` are free-form prose anyway, so mechanical matching was always going to be unreliable.
2. Extend the hook to parse `drive.yaml`'s `stage:` field and surface it in the stderr message (the hook already has `jq` available — adding a `yq` parse or a `grep -E '^stage:'` is light).
