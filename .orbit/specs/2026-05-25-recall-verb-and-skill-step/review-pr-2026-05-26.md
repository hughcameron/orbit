## AC coverage

### ac-01 — substrate-wide fan-out verb · APPROVE

`orbit recall <topic>` is wired in `crates/cli/src/main.rs` as a top-level `Command::Recall` and routes to `VerbRequest::SubstrateRecall` via `build_request`. Implementation in `crates/core/src/verbs.rs:5829` validates non-empty topic, fans across all five types through `recall_memory` / `recall_cards` / `recall_choices` / `recall_specs` / `recall_memos`, and merges with min-max normalisation per bucket. Default ordering (`verbs.rs:5900`) sorts by score desc → type rank (`memory > choice > card > spec > memo` via `recall_type_rank` at `verbs.rs:5818`) → id asc. `RecallMatch` carries `id`, `kind`, `score`, `snippet`, `path` — the exact tuple shape ac-01 specifies. Zero-match handling: per-type buckets that return empty contribute zero rows; `normalise_min_max` short-circuits on empty buckets (`verbs.rs:5915`) so there's no division-by-zero risk. The verb-routed name is `substrate.recall`, advertised by MCP `tool_descriptors()` (`crates/mcp/src/main.rs`).

Unit test `rcall_returns_substrate_wide_results` (`verbs.rs:14185`) seeds one matching artefact per type and asserts all five `kind`s appear, plus per-type score normalisation collapses single-match buckets to 1.0. Live smoke confirmed 4 of 5 types against repo substrate.

### ac-02 — --type filter · APPROVE

CLI parses `--type memory,choice` as a comma list, trimming and dropping empty tokens (`main.rs:1501-1518`), then forwards a `Vec<String>` to the verb. Verb layer (`verbs.rs:5843-5867`) lowercases, treats empty list as default (all-five), validates each token against `RECALL_TYPES`, and emits `Error::malformed` for any invalid token. The error message names the offending token (`"invalid type 'bogus'"`) and lists all valid values, matching the spec's canonical-error contract.

`rcall_filter_by_type` (`verbs.rs:14229`) covers all four ac-02 cases: explicit narrowing, empty-list-as-default, invalid-token rejection, mixed valid+invalid rejection.

### ac-03 — JSON envelope parity · APPROVE

`rcall_json_envelope_parity` (`parity.rs:1187`) is the load-bearing test. It populates a substrate, builds the canonical envelope via `envelope_ok_string(execute(...))`, then runs MCP `tools/call` with `name: "substrate.recall"` and asserts byte-identical envelope text — covering both default and `--type`-filtered fan-out. Also verifies `tools/list` advertises `substrate.recall`. Human-readable output is rendered by `render_substrate_recall` (`main.rs:1958-1976`) with a default tabular form.
