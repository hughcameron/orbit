## Findings

### [LOW] Warning sink not added; ac-07 audit data-quality risk carried forward
**Category:** test-gap
**Description:** The v3 spec-review flagged that ac-07's audit memo asks for a per-file-kind warning-fire tabulation but ac-02 doesn't pin a durable log sink — hook stderr goes to the session transcript, not a grep-able cross-session file. The implementation matches the spec as written (hook calls `echo ... >&2`, no append-to-log), so this isn't an implementation gap. But it carries forward the same risk: at week 4 the audit will need to derive the count from session transcripts rather than read a `.orbit/.code-investigate-warnings` file. The v3 reviewer noted this does not block close (observation-band).
**Evidence:** `grep -n "code-investigate-warnings" plugins/orb/hooks/code-investigate-nudge.sh` returns no hits; the warning sink the v3-LOW-2 suggested was not added.
**Recommendation:** Before the 4-week audit window opens (2026-06-15), add a one-line append to `.orbit/.code-investigate-warnings` in the hook (timestamp, session-id, file path, file-kind classifier), with matching `.gitignore` entry. Tracked as a known follow-up on the spec, not an implementation defect.

### [LOW] Mark-script appends without dedupe
**Category:** edge-case
**Description:** Running `code-investigate-mark.sh file src/c.rs` twice produces two identical entries in the marker. The hook's matching is exact-path so extra entries are harmless for matching, but the marker grows unboundedly across a long session. Idempotency-by-dedupe would keep the file readable for the audit step and avoid pathological growth in long-running sessions.
**Evidence:** edge-5 probe — after one `mark file src/c.rs` followed by two repeats, the marker had 3 entries for the same path.
**Recommendation:** Optional tightening — skip the append if the latest entry has the same `<kind>\t<path>`. Not a defect; the contract doesn't require dedupe and matching is unaffected.

### [INFO] Two pre-existing test failures in the bash test suite
**Category:** environment-mismatch
**Description:** `plugins/orb/scripts/tests/test-promote-gate-propagation.sh` and `test-sidecar-layout.sh` both fail at the "promote.sh: orbit spec create did not produce expected file" step. Neither test touches a file modified in this PR's diff (`promote.sh` and sidecar logic are unchanged). The failures reproduce on the same tree regardless of whether the code-investigate changes are present, so they pre-date this branch.
**Evidence:** Reproduced with current HEAD; `git diff main~3...HEAD -- plugins/orb/scripts/promote.sh plugins/orb/scripts/tests/` shows zero changes to those files.
**Recommendation:** Not blocking this PR. Flag for the next session — likely a `promote.sh` ↔ `orbit spec create` filename-shape drift unrelated to this work.
