## Findings

### [LOW] ac-09 sub-clause: setup SKILL.md doesn't cite the renamed card slug

**Category:** test-gap (spec-verification miss, not a behavioural defect)

**Description:** Spec ac-09 verification names *four* SKILL.md files that should cite `0026-agent-prose-discipline`: design, review-spec, review-pr, setup. Only the first three carry the citation. Setup SKILL.md was edited as part of the cascade — the pre-state mentioned "the BLUF prose contract" inline (no slug) and the post-state reframes the canonical files as METHOD.md + STYLE.md without a card-slug reference.

**Evidence:**
```
$ grep -rln '0026-agent-prose-discipline' plugins/orb/skills/
plugins/orb/skills/review-spec/SKILL.md
plugins/orb/skills/design/SKILL.md
plugins/orb/skills/review-pr/SKILL.md
$ grep -c '0026' plugins/orb/skills/setup/SKILL.md
0
```

**Recommendation:** Add a one-line pointer to setup/SKILL.md alongside the canonical-files description — e.g. at line 151, after introducing METHOD.md and STYLE.md, append "Card 0026 (`.orbit/cards/0026-agent-prose-discipline.yaml`) owns the prose discipline." This is a one-line addition. Alternatively, treat the rewrite as semantically satisfying the AC (the setup skill now points at STYLE.md, which is the renamed-card's canonical surface) and update the spec verification to reflect a 3-file cascade. Either path closes the gap; the current state is a defensible but strictly-out-of-contract reading.

**Severity rationale:** LOW — no behavioural defect, no user-visible regression, no test failure. The cascade landed substantively; only the bookkeeping line is missing. Not a merge blocker.

### [INFO] Verbs.rs pin-suppression test fixture only exercises METHOD.md drift

**Category:** test-coverage observation

**Description:** `conformance_pin_behind_fires_and_suppresses_per_file` and `conformance_pin_ahead_fires_and_suppresses_per_file` (verbs.rs:10138, 10162) write only METHOD.md as the drift fixture; the test then asserts NO `byte_drift` or `missing` findings exist anywhere in the result, which transitively confirms that STYLE.md missing-state IS suppressed under pin-behind. The suppression is enforced through the unified `CANONICAL_FILES` loop, so the existing assertion shape is sound — but the test fixture doesn't dirty STYLE.md explicitly. A future regression that broke pin-suppression for one file but not the other would slip past these two tests.

**Evidence:** `verbs.rs:10141-10148` writes only `METHOD.md` as hand-edited; STYLE.md is absent (would trigger a `missing` finding under unpinned/matches). The all-or-nothing assertion `all(|f| f.state != "byte_drift" && f.state != "missing")` covers STYLE.md missing-state via the loop, so this is belt-and-braces, not a real gap.

**Recommendation:** No change required. If future churn touches the suppression path, consider adding a STYLE.md-specific pin-suppression fixture to harden the test surface. Not a merge blocker.
