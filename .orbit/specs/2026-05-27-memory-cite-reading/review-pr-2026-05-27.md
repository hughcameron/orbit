The legacy-fixture test plants a YAML file without the `cites:` key, parses it, re-serialises, and asserts the bytes are unchanged. No on-disk SQLite migration was needed (per the cycle-1 substrate-framing correction); the YAML round-trip is the only contract. Verified.
- **ac-02 wire-shape projection.** `MemoryMatch.memory` uses `#[serde(serialize_with = "serialize_memory_for_match")]` (verbs.rs:1327). The custom serializer (verbs.rs:1344-1359) emits all 5 fields explicitly — including an empty `cites` array — so callers can detect cite presence without parsing the body. This resolves the tension flagged in the request: the on-disk Memory omits empty `cites:` (byte-identity), but the wire shape always carries it (detectability). Verified.
- **ac-04 second-pass placement.** The new loop (verbs.rs:2406-2477) runs AFTER the existing corpus-vs-considered loop at verbs.rs:2358-2404 and does NOT touch it. The new loop iterates `spec.memories_considered` (the existing loop iterates `memory_match` corpus results). The refusal message names the memory key, the missing cite paths, and the actionable next step. `--force` lands bypassed entries on `SpecCloseResult.forced_uncited_evidence`. Verified.
- **ac-07 regression-suite design.** The test (verbs.rs:10625-10742) walks `CARGO_MANIFEST_DIR/../../../.orbit/memories`, asserts every YAML parses, asserts every loaded memory has empty `cites`, then plants each memory into a tempdir layout to drive `spec.close`'s cite-evidence gate. Skips with `eprintln!` when the path isn't reachable (test running outside the dev checkout) — appropriate for a local regression detector. Verified.

## Findings

### [INFO] `serialize_memory_for_match` hardcodes the 5-field projection
**Category:** maintainability
**Description:** `verbs.rs:1344-1359` enumerates Memory's fields by name (key, body, timestamp, labels, cites) inside a custom serializer. If `Memory` later acquires another field, the projection silently drops it from the wire shape. Tests for ac-02 (`cite_match_exposes_cites_on_each_result` and the two parity tests) assert cites is present but don't assert the projection's full field set.
**Evidence:** `orbit-state/crates/core/src/verbs.rs:1344-1359` — `state.serialize_field("key", ...)` through `state.serialize_field("cites", ...)` with hardcoded count `5`. No drift-detection test.
**Recommendation:** Not a blocker. A future hardening would add a test that asserts `serde_json::to_value(MemoryMatch { ... })` contains exactly the field set on `Memory`, or alternatively use a `#[serde(flatten)]` proxy struct with a `cites_present` shim. Leave as INFO; the current scope ships clean.

### [INFO] `forced_uncited_evidence` carries formatted sentences; sibling `forced_unreconciled` carries bare keys
**Category:** wire-shape consistency
**Description:** `SpecCloseResult.forced_uncited_evidence: Vec<String>` (verbs.rs:1169-1177) stores entries shaped like `"memory 'X' missing cite_evidence for: a.md, b.md — read a.md, b.md and record an excerpt in cite_evidence"`. The sibling `forced_unreconciled` (same struct) carries bare memory keys. A downstream consumer parsing the response gets prose for one field and identifiers for the other.
**Evidence:** verbs.rs:2453-2459 (refusal-message formatting reused as the bypass record); compare to verbs.rs:2400-2404 (`forced_unreconciled` is built from bare keys).
**Recommendation:** Not a blocker — both fields are documented and the bypass case is the unusual path. A future cleanup could split refusal-message-prose from the bypass-record-shape so the response carries structured `{key, missing_paths: Vec<String>}` entries. Leave as INFO.

### [INFO] Considered-memory-not-on-disk path silently skips the cite-evidence gate
**Category:** edge-case
**Description:** verbs.rs:2419-2426 — when `memories_considered[i].key` references a memory file that does not exist on disk, the new pass `continue`s. The inline comment cites the existing corpus-vs-considered pass as covering the inverse direction, but the inverse direction is "matched-but-not-considered," not "considered-but-not-on-disk." A spec author who renames a memory file after reconciling it would slip past this gate.
**Evidence:** verbs.rs:2419-2426 inline comment + skip; no test exercises the missing-on-disk branch (the cite tests all plant the memory before invoking close).
**Recommendation:** Not a blocker — out of scope for this spec (cite-evidence coverage is the named invariant; existence is upstream of it). Worth noting as a future hardening candidate: an unwriteable / renamed memory key on a `memories_considered` entry is a substrate-integrity gap the existing reconciliation pass also tolerates. Leave as INFO.

## Honest Assessment

Implementation matches the spec's contract precisely. Every cycle-1 substrate concern (the SQLite-mis-framing HIGH, the ac-04 loop-collection ambiguity MEDIUM, the ac-05 fourth-directive MEDIUM, the ac-07 gate-promotion MEDIUM) is resolved in code as well as in the spec text. Both halves are tested.
